## Create application

import { Step, Steps } from 'fumadocs-ui/components/steps';

<Steps>
<Step>
### Generate project

Change `myproject` to name of you project and run following command using [Clojure CLI](https://clojure.org/guides/install_clojure):

```shell
clojure -Ttools install-latest :lib io.github.seancorfield/deps-new :as new && \
clojure -Sdeps '{:override-deps {org.clojure/clojure {:mvn/version "1.12.0"}}}' \
    -Tnew create :template io.github.abogoyavlensky/clojure-stack-lite :name myproject
```

If you already have Clojure CLI with Clojure `1.12.x` and `deps-new` tool, then you can just run:

```shell
clojure -Tnew create :template io.github.abogoyavlensky/clojure-stack-lite :name myproject
```
</Step>

<Step>
### Start development

<Callout type="info">
 [mise](https://mise.jdx.dev/getting-started.html) is just convenient tool to manage system deps in version isolation for different projects.
 If you want, you can install all required tools manually, consult `.mise.toml` for specific versions.
</Callout>

```shell
cd myproject
mise trust && mise install
bb clj-repl
user=> (reset)
```
</Step>
</Steps>

   The server should be available at [http://localhost:8000](http://localhost:8000).

![Clojure Stack Lite starter page](@/public/starter.png)

### Update Placeholders

Also you might want to edit some details in placeholders before going to production:

- `LABEL` in `Dockerfile` to point out your project's repository for correct image attribution
- domain in the `id` field of the `resources/public/manifest.json` to install your web app as a PWA
- update description at `TODO: add project description` in the `README.md`

## Project structure

The generate project will have the following structure:

```
├── .clj-kondo/            # Clojure linting configuration
├── .github/               # GitHub Actions workflows and configurations
├── .kamal/                # Kamal secrets template (only used if you use Kamal)
├── config/                # Kamal deployment configuration (only used if you use Kamal)
├── dev/                   # Development configuration directory
│   └── user.clj           # User-specific development configuration
├── resources/             # Static resources and configuration files
│   ├── public/            # Public assets (CSS, JS, images)
│   ├── migrations/        # Database migration files
│   ├── config.edn         # Main configuration file for the application
│   ├── config.dev.edn     # Development-specific configuration
│   ├── config.e2e.edn     # Test-specific configuration for end-to-end testing
│   └── logback.xml        # Logging configuration file
├── src/                   # Source code directory
│   └── {{name}}           # Main namespace directory
│       ├── core.clj       # Application entry point
│       ├── db.clj         # Database system component and main operations
│       ├── handlers.clj   # HTTP request handlers
│       ├── routes.clj     # Route definitions
│       ├── server.clj     # Server system component
│       └── views.clj      # HTML templates and components with Hiccup
├── test/                  # Test files directory
│   └── {{name}}           # Test namespace directory
│       ├── home_test.clj  # Example test for home page
│       └── test_utils.clj # Test utilities
├── .cljfmt.edn            # Formatting configuration
├── .gitignore             # Git ignore rules
├── .mise.toml             # mise-en-place configuration with system dependencies
├── bb.edn                 # Babashka tasks configuration for managing application
├── deps.edn               # Clojure dependencies and aliases
├── Dockerfile             # Dockerfile for building the application image
├── LICENSE                # License file, AGPLv3 by default, for motivation check: https://plausible.io/blog/open-source-licenses
└── README.md              # Project documentation
```
